Platform Overview

B4.U.GO is a location-based shopping reminder app. Users add items to their shopping list and associate them with stores. Using geofencing, the app reminds users to buy items exactly when they're near the relevant store.

"Never forget to buy essentials again. B4.U.GO reminds you to shop when you are near the store."
📍

Service Breakdown

🔐 auth-service

User identity, OAuth social logins, JWT token lifecycle, push notifications, analytics events.

ModulePurpose
AuthModuleGoogle / Facebook / Apple OAuth via Passport.js, JWT issuing
UsersModuleUser CRUD, profile & role management
RedisModuleSession caching, token blacklisting
PushModulePush notification registration and dispatch
AnalyticsModuleUser event tracking (app_open, item_added, etc.)
Notable: Custom Passport guards preserve the OAuth state parameter through Firebase Hosting, which strips non-__session cookies. Mobile deep-link redirect via b4ugo://auth/callback.

📦 items-service

Core business logic - shopping items, stores, maps, chains, barcodes, feedback, file uploads.

ModulePurpose
ItemsModuleShopping item CRUD, bulk ops, status management
StoresModuleStore CRUD, PostGIS geospatial nearby search (ST_Distance), Google Places caching
MapsModuleKMZ/CSV geography imports for custom store maps
ChainsModuleGlobal chain store management (VIP stores for paid chains)
BarcodeModuleUPC/EAN barcode lookup
FeedbackModuleUser-submitted store feedback/reports
UploadModuleImage uploads to GCS bucket
Notable: PostGIS geometry(Point, 4326) column with GiST index for spatial queries. PostGIS extension installed in the items schema (not public) for Prisma multi-schema compatibility.

🔍 scan-service

AI-powered store scanning/categorization, taxonomy management, analytics dashboards.

ModulePurpose
TaxonomyModuleCategory & keyword management (multilingual: en/he), Google Product Taxonomy mapping
RedisModuleAI categorization result caching, scan metrics
AuthModuleJWT verification
Notable: Uses Google Gemini AI for intelligent store categorization. Google Maps Places API for store discovery. Scan results cached in Redis to reduce API costs.

Database Design

Schema Architecture

Single PostgreSQL 16 instance with multiple schemas - each service connects via Prisma with its own isolated schema (bounded context):

Prisma ConfigDB SchemaService
auth-service/prisma/schema.prismaauthauth-service
items-service/prisma/schema.prismaitems + publicitems-service
scan-service/prisma/schema.prismascanscan-service

Core Entities

👤 User
  • uuid id PK
  • string email UK
  • enum role USER | ADMIN
  • enum status ACTIVE | DISABLED | BANNED
  • enum subscription FREE | PRO | PLUS
📝 Item
  • uuid id PK
  • string name
  • enum status ACTIVE | DONE | DELETED
  • int quantity
  • string barcode
  • uuid mapId FK
🏪 Store
  • uuid id PK
  • string name
  • enum type PERSONAL | GLOBAL | GOOGLE
  • geometry location PostGIS Point 4326
  • string placeId UK
  • string[] categories
🗂️ Category
  • uuid id PK
  • string name UK
  • string[] googleTypes
  • int taxonomyId
Cross-Service Decoupling: items-service and scan-service store userId as a plain string - no foreign key to the auth schema. Auth-service is the single source of truth for user identity.

Key Architectural Decisions

Multi-schema Single DB

Each service has its own Prisma schema (auth, items, scan) as bounded contexts, sharing one PostgreSQL instance for cost efficiency.

Firebase Hosting as Ingress

Global CDN + SSL + path routing - needed because me-west1 doesn't support direct Cloud Run domain mappings.

Shared JWT Secret

All backend services verify tokens independently without inter-service calls.

userId as Plain String

Items/scan services reference users by ID string (no FK) - intentional decoupling from auth schema.

PostGIS in items Schema

Required for Prisma's multiSchema feature to keep spatial functions in the working schema.

Upstash over Memorystore

Serverless Redis for cost efficiency and TLS-native from Cloud Run.

Selective Deployment

git diff-based change detection deploys only affected services.

Monorepo

All services, web apps, and mobile app in one repo with scoped linting and selective CI/CD.
