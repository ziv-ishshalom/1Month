RBAC Auth Guard Stack

auth-service / items-service / scan-service
Pattern: Every admin endpoint uses a consistent 3-layer guard stack — JWT validation, role extraction, and decorator-driven authorization. 70 endpoints were secured in a single RBAC remediation pass across all 3 services. 
// roles.decorator.ts — Custom decorator for declarative role requirements
export const ROLES_KEY = 'roles';
export const Roles = (...roles: string[]) =>
  SetMetadata(ROLES_KEY, roles);

// roles.guard.ts — Reflector-based guard reads @Roles() metadata
@Injectable()
export class RolesGuard implements CanActivate {
  constructor(private reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean {
    const required = this.reflector.getAllAndOverride<string[]>(
      ROLES_KEY, [context.getHandler(), context.getClass()]
    );
    if (!required) return true;
    const { user } = context.switchToHttp().getRequest();
    return required.includes(user.role);
  }
}

// Usage: class-level guards + method-level decorator
@Controller('admin/analytics')
@UseGuards(JwtAuthGuard, RolesGuard)  // ← Guard stack on every route
@Roles('ADMIN')                      // ← Declarative: only ADMIN role
export class AnalyticsController {
  @Get('stats')
  getStats() { ... }    // Inherits class-level guards
}
💡 Why class-level? Applying guards at the controller class eliminates the risk of forgetting @UseGuards() on individual routes — a common source of privilege escalation bugs in NestJS apps.

PostGIS Spatial Scan Pipeline

scan-service / stores.service.ts
Pattern: The scan funnel resolves shopping items to nearby stores using a 5-phase pipeline: AI categorization → VIP store matching → Redis cache check → Google Places API → result caching. 
// Phase 1-2: Categorize item + check VIP stores first
const category = await this.aiCategorizer.categorize(item.text);
const vipStores = await this.prisma.$queryRaw`
  SELECT *, ST_Distance(
    location::geography,
    ST_SetSRID(ST_MakePoint(${lng}, ${lat}), 4326)::geography
  ) as distance
  FROM stores
  WHERE ST_DWithin(
    location::geography,
    ST_SetSRID(ST_MakePoint(${lng}, ${lat}), 4326)::geography,
    ${radiusMeters}            -- user's scan radius
  )
  AND is_vip = true            -- VIP stores checked first
  AND categories @> ARRAY[${category}]
  ORDER BY distance ASC
  LIMIT ${maxResults}
`;

// Phase 3: Redis cache — avoid redundant Google API calls
const cacheKey = md5(`${item.text}:${lat}:${lng}:${radius}`);
const cached = await this.redis.get(`SCAN:GOOGLE:ITEM:${cacheKey}`);
if (cached) return JSON.parse(cached);

// Phase 4-5: Google Places API → cache for future users
const results = await this.fetchGooglePlaces(item, category, lat, lng);
await this.redis.set(cacheKey, JSON.stringify(results), 'EX', 86400);
return results;
💡 Why VIP-first? Partner stores bypass the Google API entirely — reducing API costs and guaranteeing instant results. The ST_DWithin spatial index makes the PostGIS query sub-millisecond.

7-Gate Notification State Machine

mobile / backgroundTaskLogic.ts
Pattern: Background scan notifications pass through 7 strict gates to prevent spam. All decision logic lives in pure functions (no Expo APIs) — enabling 47 deterministic unit tests. 
// evaluateNotificationGates() — Pure function, fully unit-tested
export function evaluateNotificationGates(params: GateParams): GateResult {
  // Gate 3: Spatial cooldown — same stores + haven't moved enough?
  const fingerprint = stores.map(s => s.id).sort().join(',');
  if (fingerprint === lastFingerprint) {
    const moved = calculateDistance(lat, lng, lastLat, lastLng);
    if (moved < minRefreshDistance)
      return { gate: 3, action: 'skip', reason: 'spatial_cooldown' };
  }

  // Gate 4: Time dedup — same stores + too recent?
  if (fingerprint === lastFingerprint) {
    const elapsed = (Date.now() - lastNotifTime) / 60000;
    if (elapsed < sameStoreCooldownMin)
      return { gate: 4, action: 'skip', reason: 'time_dedup' };
  }

  // Gate 5: Pause — user manually paused? Stop GPS entirely
  if (notificationsPaused)
    return { gate: 5, action: 'stop_gps', reason: 'paused' };

  // Gate 6: Quiet Hours — scan runs, notification suppressed
  if (isInQuietHours(quietConfig))
    return { gate: 6, action: 'skip', reason: 'quiet_hours' };

  // Gate 7: All gates passed → notify user
  return { gate: 7, action: 'notify', fingerprint, lat, lng };
}
💡 Why pure functions? The 572-line backgroundTask.ts orchestrator handles side effects (storage, GPS, notifications). All decisions are delegated to backgroundTaskLogic.ts — testable without mocking Expo APIs, React Native, or AsyncStorage.

Smart Driving — 3-Tier Hysteresis

mobile / backgroundTaskLogic.ts
Pattern: GPS speed automatically adjusts the scan radius across 3 tiers. Upgrades require N high-speed events in a rolling window. Reverts require consecutive low-speed readings (hysteresis) to prevent ping-ponging.
Smart Driving State Machine
🚶 Pedestrian User's radius (e.g. 100m) distanceInterval: 20m
>15 km/h × 3 events →
← <7 km/h × 2 consecutive
🚗 City Driving 500m refresh radius distanceInterval: 100m
>60 km/h × 3 events →
← <40 km/h × 2 consecutive
🛣️ Commuter 2000m refresh radius distanceInterval: 500m 
// checkPedestrianToCityUpgrade() — Pure, unit-tested
export function checkPedestrianToCityUpgrade(
  speedKmh: number,
  timestamps: number[],
  settings: ServerSettings
): { upgrade: boolean; timestamps: number[] } {
  const now = Date.now();
  const windowMs = settings.drivingNotifWindowMin * 60000;

  // Filter to events within the rolling window
  const recent = timestamps.filter(t => now - t < windowMs);

  if (speedKmh > settings.drivingSpeedKmh) {
    recent.push(now);
    if (recent.length >= settings.drivingNotifCount) {
      return { upgrade: true, timestamps: [] }; // Reset on upgrade
    }
  }
  return { upgrade: false, timestamps: recent };
}

// checkCommuterToCityRevert() — Hysteresis prevents ping-pong
export function checkCommuterToCityRevert(
  speedKmh: number,
  revertTimestamps: number[],
  settings: ServerSettings
): { revert: boolean; timestamps: number[] } {
  if (speedKmh >= settings.commuterRevertSpeedKmh) {
    return { revert: false, timestamps: [] }; // Reset counter
  }
  const updated = [...revertTimestamps, Date.now()];
  return {
    revert: updated.length >= settings.commuterRevertCount,
    timestamps: updated
  };
}
💡 Why hysteresis? Without consecutive-reading requirements, a single slow GPS reading at a traffic light would revert Commuter → City, then immediately re-upgrade — causing notification spam and GPS restart thrashing. Server-controlled thresholds allow tuning without app updates.

AI Moderation Pipeline

CityFix / reports-service / upload.interceptor.ts
Pattern: Every civic report passes through a 5-stage AI pipeline synchronously at upload time. Bad content never reaches the database. Runs on-device TensorFlow.js + Tesseract.js OCR inside a NestJS interceptor. 
// 5-Stage Report Upload Pipeline (NestJS Interceptor)

// Stage 1: Text Moderation — reject abusive descriptions
const hasProfanity = leoProfanity.check(dto.title + ' ' + dto.description);
if (hasProfanity)
  throw new BadRequestException('Content contains inappropriate language');

// Stage 2: Image Optimization — resize + WebP conversion
const optimized = await sharp(file.buffer)
  .resize({ width: 1200, withoutEnlargement: true })
  .webp({ quality: 80 })
  .toBuffer();

// Stage 3: OCR PII Detection — scan for license plates, IDs
const ocrResult = await Tesseract.recognize(optimized, 'eng+heb');
const piiRegions = ocrResult.data.words
  .filter(w => w.confidence > 60)
  .map(w => w.bbox);  // Bounding boxes of detected text

// Stage 3b: Blur PII regions using Sharp composite
let processed = optimized;
for (const box of piiRegions) {
  processed = await sharp(processed)
    .extract({ left: box.x0, top: box.y0,
              width: box.x1 - box.x0, height: box.y1 - box.y0 })
    .blur(20)  // Heavy Gaussian blur on PII region
    .toBuffer();
}

// Stage 4: NSFW Content Detection — TensorFlow.js
const predictions = await nsfwModel.classify(processed);
const nsfw = predictions.find(p =>
  p.className === 'Porn' && p.probability > 0.7
);
if (nsfw) throw new BadRequestException('Inappropriate image');

// Stage 5: Commit — safe, blurred, optimized WebP → disk + DB
await fs.writeFile(uploadPath, processed);
return await this.prisma.report.create({ data: { ...dto, imageUrl } });
💡 Why synchronous? Processing at upload time (not via a queue) means bad content is never persisted. Trade-off: NSFWJS pulls heavy TensorFlow node bindings — container memory must be ≥1.5 GB to avoid OOM under load.

OAuth State Guards — Firebase Cookie Fix

auth-service / guards
Pattern: Firebase Hosting strips all cookies except __session. Standard Passport.js session-based state verification breaks. Custom guards explicitly pass OAuth state through query params instead of relying on cookies. 
// Problem: Firebase Hosting strips session cookies
// → Passport's OAuth state verification fails
// Solution: Custom guard passes state via query params

@Injectable()
export class GoogleStateAuthGuard extends AuthGuard('google') {
  getAuthenticateOptions(context: ExecutionContext) {
    const request = context.switchToHttp().getRequest();
    const state = request.query.state;
    return {
      state,                          // Pass to Google OAuth
      accessType: 'offline',           // Refresh tokens
      prompt: 'consent',               // Force consent screen
    };
  }
}

// Callback: state tells us where the request originated
@Get('google/callback')
@UseGuards(AuthGuard('google'))
async googleCallback(@Req() req, @Res() res) {
  const state = JSON.parse(req.query.state || '{}');

  if (state.isMobile) {
    // Mobile app → deep link with tokens
    return res.redirect(
      `b4ugo://auth/callback?token=${token}`
    );
  }
  // Admin web → set __session cookie + redirect
  res.cookie('__session', token, { httpOnly: true });
  return res.redirect('/dashboard');
}
💡 Why this matters: Without this guard, the backend can't distinguish mobile from web OAuth flows — it would redirect a mobile user to a browser dashboard or set a cookie the app can't read. The state param carries isMobile and redirectUri through the entire OAuth round-trip.
